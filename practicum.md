<details>
<summary>API</summary>

* Количество, суммы, страницы, размеры проверять границами и эквивалентами (Положительное, 0, отрицательное)
* Всегда проверять статусы ответов (Ошибка? Если да, то какая? Если успех, то какой? 200 или 201?)
* Проверять текст ошибки или сообщение после успеха.
  
  * **Примеры**:
  
```
Status: 400 Bad request  
{
"message”:"unknown user”
}
```

```
Status: 200 OK 
{
"message”:"QR created successfull”,
"qr_link”:”https://www.qrlink.com/jhadsjfhasjkfhjasdfjhsbdfhjsdbvjfbvhjs”
}
```
  
* Проверять набор параметров
  
    * **Позитивные:**
      - полный набор
      - обязательные поля
      - Проверить, как подставляются данные по умолчанию
      - Проверить, что каждый раз при POST генерится новый уникальный QR? Если нет, то уточнять у аналитика как должно работать?



    * **Негативные:**
      - Некорректные значения в полях? (Несуществующий размер? Размер, в котором ничего не разглядишь: отрицательные значения размеров, значения, которые не ожидает сервер)
      - Без параметров (Без одного обязательного, Без всех)
      - Там, где можно передать пустое значение, проверить с отсутствием поля вообще.
      - Добавить поле не по доке. что будет?
      - Подменить метод. Что будет?
